Storage

Context Pilot stores all data in a local SQLite database. The data directory is already configured.

📁
Data directory:
~/.contextpilot
Set via CONTEXTPILOT_DATA_DIR env variable or defaults to ~/.contextpilot
🗂
What's stored here:
SQLite databases (memories, FTS index, embeddings), profile configs, connector settings, folder sources, webhook configs. Everything is file-based — easy to back up and migrate.
👤

Your First Profile

Profiles keep your data completely isolated. Each profile gets its own database, connectors, and folder sources. You can create more later.

The "default" profile already exists. Enter a new name to create an additional one.
💡
What's isolated per profile:
Memories & tags, connector configs, folder sources, webhooks, embeddings, templates, and the knowledge graph. Switch profiles instantly from the header dropdown.
🔌

Context Assembler

The assembler intelligently selects and compresses memories to fit within a token budget — perfect for feeding context to AI models.

📚Memories
➞
🎯Relevance
➞
📦Compress
➞
🚀Context
💡
How it works:
1. Memories are scored by relevance (tags, recency, usage)
2. Top memories are selected within your token budget
3. Smart compressors (bullet, code, YAML, mermaid) shrink content
4. The result is a perfectly sized context block for your AI
📚
